# 연구 요약 — 재즈 피아노 스타일 파인튜닝의 병목 진단 (D0→D4)

> 이 문서 하나로 지금까지의 연구 전체를 이해할 수 있도록 정리했습니다.
> 각 실험의 정본은 `docs/dN_experiment/DN_RESULTS.md`, 층별 로드맵은
> [`RESEARCH_ROADMAP.md`](RESEARCH_ROADMAP.md)에 있습니다.

## 한 줄 요약

작은(13.4M) Music Transformer를 특정 재즈 피아니스트 스타일로 파인튜닝했을 때 생긴
품질 저하를, **하나의 실험이 하나의 질문에 답하는 통제 실험 5단계(D0→D4)** 로 역추적해
원인을 층별로 분리하고 각각 처방을 확정한 기록입니다. 핵심 결론:

> **품질 저하는 base 모델의 표현력 한계가 아니었다.**
> 병목은 (L2) 데이터 레짐, (L3) 파인튜닝 방식, (L1) 디코딩 문법, (L4) 조건 프라이머
> 텍스처 — 이 네 층에 나뉘어 있었고, 각각 독립적으로 해결 가능했다.

## 배경 — 무엇을, 왜

- **목표 시스템**: 인간 재즈 피아니스트와 실시간으로 주고받는 AI 즉흥연주(call-and-response).
- **이번 연구의 범위**: 그 첫 벽돌인 **심볼릭(MIDI) 생성 모델의 스타일 개인화**.
  대규모 재즈 코퍼스로 학습한 모델을 특정 연주자 스타일로 좁히려 하자 생성이
  단조로워지고 다양성이 붕괴했다. "base가 작아서 안 되는 것"인지, 아니면 다른
  원인인지가 이 연구의 출발 질문이었다.
- **모델**: 밑바닥부터 학습한 Music Transformer (n_layers=6, d_model=512, ~13.4M params,
  legacy performance 토큰: NOTE_ON/OFF/TIME_SHIFT/VELOCITY).
- **데이터**: 솔로 재즈 피아노 MIDI 2,777곡(스튜디오 1,994 + 라이브 783).
- **하드웨어**: 로컬 Apple Silicon Mac (MPS), GPU 없음. 모든 실험을 로컬에서 수행.

## 방법론 원칙

1. **판정 기준을 실행 전에 사전 등록한다** (강한성공/부분/실패 문턱을 미리 못 박음).
2. **하나의 실험은 하나의 변수만 바꾼다** — 인과 주장이 가능하도록.
3. **하위 층이 깨진 채로 상위 층 지표를 믿지 않는다** (문법이 깨지면 다양성 지표 무의미).
4. **자기 가설도 증거가 반대하면 기각한다** — 아래 기각된 가설 목록이 그 기록.

주요 평가 지표는 `scripts/diversity_metrics.py`로 생성 MIDI에서 직접 측정한
**pooled 고유 보이싱 수**(다양성)와 **mean voicing size**(동시음 수 = 화음 두께)다.

## 진단 체인

### D0 [L2] — 병목은 데이터 레짐이다
- **질문**: 다양성 붕괴가 base 크기 탓인가, 데이터 탓인가?
- **방법**: 동일 모델·동일 하이퍼파라미터로 데이터만 16조각 → 2,777조각으로 확대.
- **결과**: val loss 5.06 → 3.19 (perplexity 6.6배 개선), 생성 다양성 전 지표 회복.
  결정타 — 16조각 모델은 이미 **자기 학습 데이터 다양성 상한의 93%** 에 도달해 있었다.
  모델이 못한 게 아니라 데이터 천장이 낮았던 것.
- **정본**: [`docs/d0_experiment/D0_RESULTS.md`](d0_experiment/D0_RESULTS.md)

### D1 [L3] — 붕괴 원인은 데이터 양이 아니라 분포 이탈
- **질문**: 스타일 적응 시 붕괴를 막는 파인튜닝 방식은?
- **방법**: 5개 팔 비교 — 스타일만 직접학습 / 사전학습만 / full FT / **LoRA 적응** / full FT+증강.
- **결과**:

  | 방식 | pooled 고유 보이싱 | 판정 |
  |---|---|---|
  | 스타일 데이터만 직접 학습 | 81 | 붕괴 |
  | 사전학습만 (상한 기준) | 145 | — |
  | 사전학습 → full FT | 108 | 부분 |
  | 사전학습 → full FT + 증강 ×12 | 80 | 재붕괴 |
  | **사전학습 → LoRA 적응** | **142 (상한의 98%)** | **채택** |

  붕괴의 원인은 데이터 양이 아니라 **사전학습 분포로부터의 이탈량**. base를 동결하는
  LoRA가 구조적 해법 — 연주자별 어댑터를 같은 base 위에 스왑하는 확장도 여기서 나온다.
- **정본**: [`docs/d1_experiment/D1_RESULTS.md`](d1_experiment/D1_RESULTS.md)

### D2 [L1] — 문법 제약 디코딩
- **질문**: 생성물의 깨진 MIDI 문법(고아 note_off 등)을 어떻게 없애나?
- **방법**: 디코딩 시 유효하지 않은 토큰의 logit을 마스킹(`--grammar_mask`, 기본 off로
  D0/D1 재현성 보존).
- **결과**: 고아 note_off 32 → **0**, 노트 밀도 +40%, 다양성 149로 유지.
  단, mean voicing size는 1.34 → 1.31로 불변 → **단선율 문제는 디코딩 탓이 아니다**(H-D3b 기각).
- **정본**: [`docs/d2_experiment/D2_RESULTS.md`](d2_experiment/D2_RESULTS.md)

### D3 [L4] — 단선율의 진짜 원인: 조건 프라이머 텍스처 잠김
- **질문**: 생성이 왜 화음이 아니라 단선율에 머무는가? (4개 가설 순차 검증)
- **기각된 가설** (정직 기록):
  - H-D3b(디코딩) — D2에서 기각.
  - H-D3d(보수적 샘플링) — 기각, temp↓ 할수록 오히려 더 단선율.
  - H-D3c(사전학습 부족) — 기각. 16ep 연장으로 val loss는 하강(3.15→3.00)했는데
    voicing은 완전 불변(1.305→1.306) → 화음 생성 능력은 이미 있었다.
  - H-D3a(데이터 lead 분리) — 필요조건이나 단독 처방 불충분.
- **입증된 원인 H-D3e**: 같은 base에서 **조건만 무프라이머로 바꾸자** voicing
  1.305 → **1.618** (원본 상한 1.657 근접). lead-only(오른손 단선율) 프라이머가
  모델을 그 텍스처에 가두고 있었다.
- **주의 — 처방은 "무프라이머"가 아님**: 조건 제어(L5)는 최종 목표에 필수. 다음 단계는
  프라이머 *포맷*을 재설계하는 것.
- **정본**: [`docs/d3_experiment/D3_RESULTS.md`](d3_experiment/D3_RESULTS.md)

### D4 [L4→L5] — 프라이머 텍스처가 생성 텍스처를 제어한다
- **질문**: 프라이머를 두껍게 만들면 생성이 두꺼워지는가? (재학습 없이 조건 MIDI만 교체)
- **결과**:

  | 팔 | 프라이머 voicing | 생성 voicing | Δ(base) | 판정 |
  |---|---|---|---|---|
  | base (lead 저음) | ~1.23 | 1.306 | — | D3 하한(1.305) 재현 |
  | twohand (두손) | ~1.43 | 1.436 | +0.130 | 부분 |
  | **chordal (화음)** | ~1.83 | **1.784** | +0.478 | **강한 성공** |
  | none (무프라이머) | — | 1.625 | +0.319 | D3 상한(1.618) 재현 |

  - 입력→출력이 **거의 선형**(1.23/1.43/1.83 → 1.31/1.44/1.78). 모델은 프라이머
    텍스처를 그대로 복사한다. 변수는 "두 손이냐"가 아니라 "프라이머가 얼마나 두꺼우냐".
  - chordal은 무프라이머 상한·원본 천장(1.657)마저 초과 → 조건 프라이머는 병목 해소를
    넘어 **텍스처 조향 수단**.
  - 대조군 두 개가 D3 좌표계를 거의 정확히 재현 → 실험이 D3와 인과 연속임을 보증.
- **처방(L5)**: 실시간 세션에서 conditioning 프라이머는 인간 입력의 실제 텍스처를
  그대로 전달한다(단선율 요약 금지). 프라이머가 곧 텍스처 리모컨.
- **정본**: [`docs/d4_experiment/D4_RESULTS.md`](d4_experiment/D4_RESULTS.md)

## 표준 레시피 (현재 확정)

1. **사전학습**: 전체 재즈 코퍼스(2,777)로 모델 전체 학습.
2. **LoRA 적응**: base 동결, 스타일 서브셋으로 LoRA 어댑터만 학습
   (`train_qlora.py --checkpoint <ckpt>`, `--train_full_model` 없이).
3. **생성**: `--grammar_mask`로 문법 보장, 조건 프라이머는 원하는 텍스처(두께)를 담아 전달.

## 남은 일 (다음 층)

- **L5 조건 제어 통합**: chord/BPM/energy 조건 준수 (Stage B 자산 이식, D5).
- **L6 실시간성**: 블록 단위 생성 + 슬라이딩 컨텍스트로 저지연 (LMDM 참조, D6).
- **L7 상호작용**: 인간 입력 실시간 반영 co-improvisation (D7).
- **보강**: 다중 시드 재현, 정성/청취 평가.

```bash

bash scripts/run_d4_experiment.sh     # D4 (STAGE=primers|gen|measure 단계 지원)
```

토큰화·디바이스·MPS 학습 절차는 [`../MPS_TRAINING_GUIDE.md`](../MPS_TRAINING_GUIDE.md) 참조.
