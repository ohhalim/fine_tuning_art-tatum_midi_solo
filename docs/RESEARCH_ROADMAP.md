# Research Roadmap — 최종 목표와 가설 기반 실험 계획

> 작성: 2026-07-12. CORE_PLAN.md(2026-06-11)의 최종 목표를 계승하되,
> D0/D1 실험 결과를 반영해 "가설 → 통제 실험 → 사전 등록 판정" 체계로 재구성한 기준 문서.
> 개별 실험의 상세 설계는 각 `docs/d*_experiment/` 문서가 정본이다.

## 1. 최종 목표

**실시간 AI 음악 협연(co-improvisation) 시스템** — 인간 연주자(또는 groove/코드 진행)와
상호작용하며 재즈 피아노 솔로를 실시간으로 생성하는 symbolic MIDI 모델.

중간 형태(CORE_PLAN 계승): BPM·코드 진행·energy를 입력받아 1–2마디 재즈 피아노 솔로
MIDI를 내는 생성기 (DAW/라이브 컨트롤러에서 사용). raw audio 생성이 아니며,
generic 재즈 base 위에 연주자별 스타일(LoRA 어댑터)을 얹는 구조.

## 2. 능력 스택 — 최종 목표에서 역산한 필요 능력

아래 순서는 의존 관계다. 하위가 깨져 있으면 상위 실험의 측정이 오염된다.

| 층 | 능력 | 상태 | 담당 실험 |
|---|---|---|---|
| L1 | **토큰 문법**: 고아 이벤트 없는 valid MIDI 생성 | 🔄 진행 중 | D2 (#1455) |
| L2 | **다양성**: 붕괴 없는 생성 분포 | ✅ 해결 | D0 (#1444), D1 (#1449) |
| L3 | **스타일**: 연주자별 적응 (다양성 유지하며) | ✅ 레시피 확정, 확장 여지 | D1 Arm D → D4 |
| L4 | **음악성**: 화성 밀도(단선율 해소), 프레이즈 구조, 코드 정합 | ❌ 미해결 | D3 (계획) |
| L5 | **조건 제어**: chord/BPM/energy 조건 준수 | ⏸ Stage B 자산 보유, 미통합 | D5 (계획) |
| L6 | **실시간성**: 블록 생성, 슬라이딩 컨텍스트, 지연 제어 | ⏸ 미착수 (LMDM 참조) | D6 (계획) |
| L7 | **상호작용**: 인간 입력의 실시간 반영 | ⏸ 미착수 | D7 (계획) |

## 3. 완료된 실험 — 확정된 사실

### D0 (#1444) — 다양성 붕괴의 원인은 데이터 레짐

- 가설 대결: base 표현력 한계(H1) vs 데이터 레짐(H2) → **H2 입증**
- 증거: 동일 모델, 데이터 16→2,777 변경만으로 val ppl 157→24, 생성 다양성 회복.
  Arm A는 자기 데이터 상한의 93%에 도달해 있었음 (모델이 아니라 데이터가 천장).
- 정본: `docs/d0_experiment/D0_RESULTS.md`

### D1 (#1449) — LoRA 커리큘럼이 붕괴를 막는다

- H-D1 ✅ 사전학습→스타일 FT 커리큘럼 유효 (LoRA로 할 때): 다양성 98% 유지 + 적응
- H-D1a ✅ LoRA ≫ full FT (142 vs 108, 적응력 동급)
- H-D1b ❌ 증강은 full FT에서 역효과 (80, 재붕괴) → 붕괴 원인은 **사전학습 분포 이탈량**
- **채택 레시피**: full-corpus 사전학습 → LoRA-only 스타일 적응 (README 명문화됨)
- 정본: `docs/d1_experiment/D1_RESULTS.md`

## 4. 진행 중 / 계획된 실험 — 가설과 판정 기준

### D2 (#1455, 진행 중) — 문법 제약 디코딩 [L1]

- **가설**: 샘플링 시 무효 토큰(고아 note_off, 중복 note_on)을 마스킹하면, 버려지던
  토큰(~노트 이벤트의 1/3)이 실제 노트로 전환되어 밀도·품질이 오르고 다양성은 유지된다.
- **실험**: D1 Arm D 체크포인트, 동일 조건 24샘플 전후 비교.
- **판정(사전 등록)**: 고아 note_off ~0 + 노트 밀도 상승 + pooled 고유 보이싱 ≥135.
- 실패 시: 마스킹이 분포를 왜곡한 것 → 기본 off 유지, 원인 분석.

### D3 (계획) — 단선율 문제: 화성 밀도 회복 [L4]

- **관찰**: 전 팔에서 mean_voicing_size 1.2–1.3 (거의 단선율). 재즈 피아노가 아님.
- **가설 후보** (D2 결과 후 우선순위 결정):
  - H-D3a: 데이터 원인 — lead 역할 분리(split_pitch 60)가 화음을 갈라놓아 학습 데이터
    자체가 단선율에 가깝다 → 학습 데이터의 voicing size 분포를 먼저 측정해 상한 확인
    (D0의 "데이터 상한" 방법론 재사용).
  - H-D3b: 디코딩 원인 — 동시 발음은 time_shift 없이 연속 note_on인데, 문법 오류/
    샘플링이 이를 깨뜨림 → D2 마스킹 후 voicing size 재측정으로 부분 검증됨.
  - H-D3c: 모델 용량/학습량 원인 → 사전학습 연장(16ep+) 후 재측정.
- **판정 기준(잠정)**: 생성 mean_voicing_size가 학습 데이터 상한의 80% 이상.

### D4 (계획) — 스타일 어댑터 확장 [L3]

- **가설**: (a) LoRA+전조 증강은 full FT와 달리 역효과가 없다 (base 동결로 이탈 메커니즘
  부재). (b) 다른 연주자 서브셋에도 같은 레시피가 재현된다 (Tatum 등 데이터 확보 시).
- **실험**: D1 하니스 재사용, LoRA 팔에 증강 데이터만 교체.
- **판정**: D1과 동일 기준 (pooled 고유 보이싱 ≥116 강한 성공).

### D5 (계획) — 조건 제어 통합 [L5]

- **가설**: Stage B의 chord-aware constrained decoding·repair 자산을 새 base(사전학습+
  LoRA) 위에 이식하면, 코드 정합(strong-beat chord-tone ratio 등 기존 지표)이 Stage B
  수준 이상으로 나온다.
- **선행 조건**: D2·D3 통과 (문법·화성 밀도가 깨진 상태에서 코드 정합 측정은 무의미).

### D6 (계획) — 실시간 블록 생성 [L6]

- **가설**: 1–2마디 블록 단위 생성 + 슬라이딩 컨텍스트로 지연 < 블록 길이를 달성하면서
  블록 경계에서 다양성·문법 지표가 열화되지 않는다.
- **참조**: LMDM의 block-wise generation, live scheduling (CORE_PLAN Long-term reference).

### D7 (계획) — 상호작용: co-improvisation [L7]

- **가설**: 최근 인간 연주 MIDI를 조건으로 넣으면(기존 conditioning 경로 재사용)
  응답 생성이 무조건 생성 대비 모티프 관련성 지표에서 우위.
- 지표 설계 자체가 선행 과제 (모티프 유사도/응답성 측정 방법 필요).

## 5. 운영 원칙

1. **판정 기준은 실행 전에 사전 등록** (D0/D1/D2 방식 유지). 실행 후 기준 변경 금지.
2. **한 실험 = 한 질문**. 팔 구성은 단일 변수 차이로.
3. 하위 층(L1–L2)이 깨진 상태에서 상위 층 지표를 신뢰하지 않는다.
4. 워크플로: 이슈(설계 포함) → 브랜치 → 구현/실행 → 결과 문서(`docs/d*_experiment/`) → PR → 머지.
5. 이 문서는 각 실험 종료 시 상태 갱신. 상세 수치는 각 결과 문서가 정본.
