# 디코더 노트 손실 진단·처방 (#1451)

> 실행: 2026-07-12, 로컬. `_merge_note`(midi_processor/processor.py)의
> 조용한 노트 손실을 진단하고 견고화. 판정 기준은 실행 전 등록.

## 질문

생성 로그의 `info removed pitch`가 **유효 음역(21–108) 안**의 pitch에도 찍힌다.
이게 (a) 데이터/인코딩 버그인가, (b) 디코더 견고성 문제인가?
그리고 이 손실이 D3의 화성 밀도(voicing) 결론을 오염시켰는가?

## 진단

### 1. 정규 데이터는 손실 0 — 데이터 버그 아님

학습 코퍼스(`data/jazz_full`) 30파일을 인코딩→디코딩 라운드트립:

| 항목 | 값 |
|---|---|
| 총 토큰 | 196,120 |
| 디코드된 노트 | 51,386 |
| orphan note_off / 재타건 / 미닫힘 | **0 / 0 / 0** |

인코더가 만든 정규 시퀀스는 문법적으로 완벽하다. 손실은 **모델 생성 시퀀스에서만** 발생한다.

### 2. 손실 경로 3가지 (생성 시퀀스, D1 Arm D, 6샘플, length 512)

| grammar_mask | 유지 노트 | 손실 노트 | 손실률 | 내역 |
|---|---|---|---|---|
| **off** | 240 | 289 | **54.6%** | orphan 145 · 재타건 53 · zero-dur 2 · **미닫힘 89** |
| **on** (D2) | 314 | 94 | **23.0%** | **미닫힘 94** (orphan·재타건 0) |

- D2의 `--grammar_mask`는 orphan note_off·중복 note_on을 **완전히 0으로** 만든다 (문서 주장 확인 ✅).
- 그러나 grammar_mask가 손대지 않는 **세 번째 경로 — 시퀀스 끝까지 note_off가 안 온 note_on** 이
  남아, 마스크를 켜도 노트의 23%가 여전히 버려진다.

### 3. D3 결론 오염 여부 — 오염 없음, 절대값만 미세 과소평가

grammar_mask ON 생성 6샘플을 원본 디코드 vs 패치 디코드로 비교:

| 디코드 | 노트 총수 | mean voicing_size |
|---|---|---|
| 원본 (손실 포함) | 299 | 1.139 |
| 패치 (손실 회복) | 397 (+32.8%) | 1.185 (+4%) |

미닫힘 손실이 화음에 **약간** 편향돼 voicing을 낮추긴 했으나 그 크기는 ~0.05.
D3의 핵심 결론(lead 프라이머 1.305 → 무프라이머 1.618, Δ≈0.31)은 이 아티팩트를
**6배 이상 압도**한다. → **D3 결론은 유효**, 절대 voicing만 소폭 과소평가됐던 것.

## 처방

`_merge_note` 견고화 (`unclosed_note_dur=0.5` 기본):

1. **재타건**(활성 pitch의 note_on 재발생): 이전 노트를 새 onset에서 닫아 **보존**
   (기존: 조용히 덮어써 손실).
2. **미닫힘 note_on**(시퀀스 끝): 폴백 duration으로 **살림** (기존: 폐기).
3. **orphan note_off**: 그대로 무시 (닫을 노트가 없으니 정상), 로그는 `verbose`로만.
4. `unclosed_note_dur=0`이면 옛 동작과 동일 — **재현 호환 보존**.

## 남은 이슈 / 다음

- `unclosed_note_dur=0.5`는 임시 상수. length를 넉넉히 두면 미닫힘 자체가 줄어드므로,
  생성 시 EOS/충분한 length 확보가 근본 처방. 폴백은 안전망.
- 생성 하니스에 `--grammar_mask` 기본화 + 패치 디코더 조합으로 D2/D3 절대 수치 재측정 여지.
