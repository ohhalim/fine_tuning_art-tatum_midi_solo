# D4 설계 — 조건 프라이머 텍스처 제어 [L5]

> **상태**: 설계 (실행 전 사전 등록)
> **로드맵**: L5 (조건 제어) — D3가 부과한 제약을 처방으로 전환
> **선행**: D3가 입증한 "조건 프라이머의 텍스처 잠김"의 처방을 검증

## 배경 — D3가 남긴 문제

D3는 단선율의 진범이 **conditioning 프라이머**임을 입증했다(무프라이머로 바꾸자
voicing 1.305→1.618, 소스 상한 1.657 근접). 하지만 "프라이머를 빼라"는 처방이 아니다 —
실시간 협연에는 조건 제어가 필수다.

**D4 착수 시 확인한 결정적 사실**: 현재 conditioning 프라이머의 텍스처가 얇다.
`prepare_role_dataset`의 `split_conditioning_target`(split_pitch=60)은
- conditioning = pitch ≤ 60 (왼손 저음부), 실측 **mean voicing 1.23**
- target = pitch > 60 (오른손 리드), 실측 mean voicing 1.16

즉 모델에게 주는 조건 자체가 거의 단선율이다. **얇은 프라이머 → 얇은 생성.**
텍스처 잠김은 방향(왼손/오른손)이 아니라 **프라이머의 화성 밀도**가 결정한다는 가설.

## 질문

조건을 유지하면서도 텍스처 잠김을 풀 수 있는가? 조건 프라이머의 화성 밀도를 높이면
생성 voicing이 회복되는가?

## 가설 (사전 등록)

- **H-D4a** — 두손/화음 포함 conditioning 프라이머(높은 voicing)를 주면, 생성 voicing이
  현재 lead 프라이머(1.305) 대비 회복된다. 얇은 프라이머가 원인이라면 두꺼운 프라이머는
  생성을 두껍게 만들어야 한다.
- **H-D4b** — conditioning을 MIDI 텍스처 대신 코드 심볼/구조 토큰으로만 주면(텍스처 정보
  없음), 모델이 텍스처를 스스로 결정해 voicing이 회복된다. (Stage B chord-aware 경로 합류)
- **H-D4-null** (귀무) — 두꺼운 프라이머를 줘도 voicing이 안 오른다면, 잠김은 프라이머
  텍스처가 아니라 다른 요인(예: role 토큰, 학습 목표 분포)에 기인.

## 팔 (arms)

동일 base 체크포인트(D3의 16ep base, `outputs/d3_experiment/armB_ext16/ckpt`)에서
**조건만** 바꾼다 — 재학습 없음, 순수 조건 변수 통제.

| 팔 | 조건 프라이머 | 목적 |
|---|---|---|
| **기준선** | lead(왼손 저음, voicing~1.23) | D3 재현 하한 (voicing 1.305) |
| **A1** | 두손 원본 앞부분(split 없음, voicing 높음) | H-D4a 검증 |
| **A2** | 화음 강조 조건(동시타만 추린 sparse voicing) | H-D4a 세부 |
| **상한** | 무프라이머 | D3 상한 (voicing 1.618) |

(H-D4b 코드심볼 조건은 토크나이저 확장이 필요 — D4가 H-D4a를 확정하면 D5로 분리)

## 판정 기준 (사전 등록 — 실행 전 고정)

주 지표: **생성 mean voicing size** (24샘플 풀링, grammar_mask=ON, 패치 디코더).
D3 좌표계에서 lead=1.305(하한), 무프라이머=1.618(상한), 소스=1.657.

| 판정 | voicing | 해석 |
|---|---|---|
| **강한 성공** | ≥ 1.55 | 조건 유지하며 상한 대부분 회복 |
| **부분 성공** | 1.45 – 1.55 | 회복하나 미완 |
| **실패** | < 1.45 | 여전히 잠김 |
| **귀무 지지** | A1 ≈ 기준선(±0.03) | 프라이머 텍스처가 원인이 아님 |

보조 지표: pooled unique voicings(하한 149 / 상한 253), note density(하한 2.58 / 상한 5.99).
단, 조건 제어가 목적이므로 무프라이머 상한 초과는 목표 아님 — **조건을 유지하면서**
하한→상한 사이를 얼마나 좁히는지가 핵심.

## 실행 (하니스)

`scripts/run_d4_experiment.sh` — D1/D3 하니스 재사용. 재학습 없이 조건별 생성→측정만.
자세한 명령은 하니스 스크립트 참조. GPU/MPS 권장(맥 터미널 MPS 실행).

## 예상 결과

프라이머 텍스처 가설이 맞다면 A1(두손 프라이머)이 강한/부분 성공. 그러면 D4의 처방은
**"조건 프라이머는 목표 텍스처의 화성 밀도를 반영해야 한다"** — 협연 시 인간 연주자의
실제 화성을 조건으로 넘기면 되므로 L5 설계에 자연 통합. 귀무가 지지되면 잠김 원인을
role 토큰/학습 목표로 다시 좁혀야 한다.
